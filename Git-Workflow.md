# Git Workflow

## Branch Strategy

main → production stable  
develop → development branch  
feature/* → fitur baru  

---

## Flow

1. Buat feature branch 
git checkout -b feature/nama-fitur


2. Commit perubahan

3. Push ke develop

4. Pull request develop → main

---

## Commit Message Convention

feat: fitur baru  
fix: bug fix  
docs: dokumentasi  
refactor: refactor code  
style: UI/CSS
